# Architecht — Draw.io → AWS Low-Level Design Generator

Turn any AWS architecture diagram drawn in **draw.io** into a deep, opinionated **Low-Level Design** document — networking, IAM, ingress/egress, CI/CD, data layer, cost estimation and optimization — generated by **Claude Sonnet 4.5 / Haiku 4.5** over **AWS Bedrock**.

![Stack](https://img.shields.io/badge/FastAPI-009688?logo=fastapi&logoColor=white) ![React](https://img.shields.io/badge/React-19-61DAFB?logo=react&logoColor=black) ![MongoDB](https://img.shields.io/badge/MongoDB-47A248?logo=mongodb&logoColor=white) ![Bedrock](https://img.shields.io/badge/AWS%20Bedrock-FF9900?logo=amazonaws&logoColor=white)

---

## Features

- 🔐 Email/password auth (JWT)
- 📥 Upload `.drawio` / `.xml` **or** paste XML directly (multi-page tabs reconciled)
- 🧠 LLD generation streamed live from **Claude Sonnet 4.5** via the AWS **Bedrock Mantle** bearer-token endpoint (swap to Haiku 4.5 via one env var)
- 🗺️ Interactive split-pane viewer — click any AWS node in the diagram or any pill to jump to its section
- 💸 Curated AWS pricing table + 18 region multipliers + on-demand refresh from the **public AWS Bulk Pricing JSON** (no AWS keys needed)
- 🔗 One-click **public, read-only share links** per LLD (revocable)
- 📄 Export to **Markdown**, **Word (.docx)** or **PDF** (browser print)
- ⚡ Resilient SSE streaming: heartbeats every ~8s + `asyncio.shield` persistence + recover-by-title fallback for ingress idle-timeouts

## Local development

### Prereqs
- Python 3.11+
- Node 20+ and Yarn 1.22+
- MongoDB running locally on `mongodb://localhost:27017` (or use Atlas)
- An **AWS Bedrock API key** with access to the chosen Claude model in the chosen region. ([Generate one →](https://console.aws.amazon.com/bedrock/home#/api-keys))

### Backend

```bash
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
# edit .env — set AWS_BEARER_TOKEN_BEDROCK and any other values
uvicorn server:app --reload --host 0.0.0.0 --port 8001
```

### Frontend

```bash
cd frontend
cp .env.example .env
yarn install
yarn start
```

Open <http://localhost:3000> — register, then go to **New LLD**.

## Deploy to Render (one click via Blueprint)

This repo ships with a [`render.yaml`](./render.yaml) blueprint. The blueprint provisions:

1. **architecht-backend** (Python web service) — FastAPI on Render's Python runtime
2. **architecht-frontend** (static site) — React build with SPA fallback rewrite

> Render does **not** host MongoDB. Spin up a free **MongoDB Atlas M0** cluster first and have the SRV connection string ready.

### Steps

1. **Push the repo to GitHub** (see "Git" below) and link it to Render.
2. In the Render dashboard click **New → Blueprint**, point it at your repo, accept the blueprint.
3. Set the **secret** env vars in the Render dashboard (the blueprint marks them `sync: false` so they aren't committed):

| Service               | Variable                     | Value                                                                  |
| --------------------- | ---------------------------- | ---------------------------------------------------------------------- |
| `architecht-backend`  | `MONGO_URL`                  | Your Atlas SRV string, e.g. `mongodb+srv://user:pass@cluster.../?...`  |
| `architecht-backend`  | `AWS_BEARER_TOKEN_BEDROCK`   | Your Bedrock API key (starts with `bedrock-api-key-`)                  |
| `architecht-backend`  | `CORS_ORIGINS`               | `https://architecht-frontend.onrender.com` (your frontend URL)         |
| `architecht-frontend` | `REACT_APP_BACKEND_URL`      | `https://architecht-backend.onrender.com` (your backend URL)           |

4. Hit **Apply**. First deploy takes ~5 minutes (backend `pip install` + frontend `yarn build`).
5. After the backend is live, copy its URL into the frontend's `REACT_APP_BACKEND_URL` and redeploy the frontend.

### Swap to Haiku 4.5

In the Render dashboard, edit `architecht-backend` → Environment, set:

```
BEDROCK_MODEL_ID=us.anthropic.claude-haiku-4-5-20251022-v1:0
```

…and save. Next request uses Haiku (much faster, ~3× cheaper).

## Endpoints (cheat sheet)

| Method | Path                              | Auth | Purpose                                                 |
| ------ | --------------------------------- | ---- | ------------------------------------------------------- |
| POST   | `/api/auth/register`              | —    | Create account                                          |
| POST   | `/api/auth/login`                 | —    | Get JWT                                                 |
| GET    | `/api/auth/me`                    | ✓    | Current user                                            |
| POST   | `/api/drawio/parse`               | ✓    | Parse XML, return pages + detected services + costs     |
| POST   | `/api/lld/generate`               | ✓    | SSE stream: meta → deltas → done(lld_id)                |
| GET    | `/api/lld`                        | ✓    | List your LLDs                                          |
| GET    | `/api/lld/:id`                    | ✓    | Get LLD                                                 |
| DELETE | `/api/lld/:id`                    | ✓    | Delete LLD                                              |
| POST   | `/api/lld/:id/share`              | ✓    | Create public share token                               |
| DELETE | `/api/lld/:id/share`              | ✓    | Revoke share                                            |
| GET    | `/api/public/lld/:token`          | —    | Public read-only LLD                                    |
| GET    | `/api/lld/:id/export/markdown`    | ✓    | Download .md                                            |
| GET    | `/api/lld/:id/export/docx`        | ✓    | Download .docx                                          |
| GET    | `/api/pricing?region=<r>`         | —    | Current per-service prices for a region                 |
| POST   | `/api/pricing/refresh?region=<r>` | ✓    | Pull live SKUs from AWS Bulk Pricing JSON               |
